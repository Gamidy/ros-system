# ROS Engineering KPI Standard V1.0

> **定义 ROS 的研发效率与质量指标体系**——自动统计每个阶段的时长、返工率、失败率、ECO 密度，
> 为 AI 优化研发流程提供数据基础。
>
> **稳定性：中** — 指标定义应长期稳定，阈值可随数据积累调整。

---

## 1. 研发全链路 KPIs

### 1.1 阶段转化指标

| 阶段链路 | 指标 | 定义 | 目标 |
|:---------|:-----|:-----|:-----|
| Idea → ProductPlan | 策划转化率 | ProductPlan 数 / Idea 数 | ≥ 30% |
| ProductPlan → Prototype | 立项率 | Prototype 数 / ProductPlan 数 | ≥ 60% |
| Prototype → Certification | 认证通过率 | Cert 通过数 / Prototype 数 | ≥ 80% |
| Certification → Mass Production | 投产率 | MP 数 / Cert 数 | ≥ 90% |

### 1.2 阶段周期指标

| 阶段 | 指标 | 定义 | 追踪方式 |
|:-----|:-----|:-----|:---------|
| Planning → P0 | 策划周期 | Plan Approved → P0 Kickoff | 自然日 |
| P0 → P2 | 开发周期 | P0 → P2 完成 | 自然日 |
| P2 → Certification | 认证周期 | P2 → Cert 获取 | 自然日 |
| Certification → MP | 投产周期 | Cert → 首批量产 | 自然日 |

### 1.3 研发效率指标

| 指标 | 定义 | 计算方式 |
|:-----|:-----|:---------|
| 单位产出周期 | 平均每个 ProductPlan 从 Idea 到 MP 的天数 | AVG(MP_Date - Idea_Date) |
| 并行项目数 | 同时处于 Development 阶段的 ProductPlan 数 | COUNT(status = Development) |
| 平台复用率 | 基于已有平台的新项目占比 | Platform_Projects / Total_Projects |

## 2. 质量指标

### 2.1 实验/测试指标

| 指标 | 定义 | 目标 |
|:-----|:-----|:-----|
| 首次通过率（FYP） | 首次测试 PASS 占比 | ≥ 70% |
| 测试覆盖率 | 实际覆盖 / 计划覆盖 | ≥ 90% |
| 缺陷逃逸率 | MP 后发现的缺陷 / 总缺陷 | ≤ 5% |
| 平均修复时间 | 缺陷创建 → 修复 | ≤ 7 天 |

### 2.2 返工指标

| 指标 | 定义 | 计算方式 |
|:-----|:-----|:---------|
| 阶段回退率 | 退回上一阶段的 Prototype 占比 | Rework_Count / Total_Prototype |
| 返工周期 | 每次回退造成的延期 | Rework_Days |
| ECO 紧急率 | 紧急 ECO 占比（非计划内） | Emergency_ECO / Total_ECO |

### 2.3 认证指标

| 指标 | 定义 | 目标 |
|:-----|:-----|:-----|
| 认证首次通过率 | 首次认证 PASS 占比 | ≥ 75% |
| 认证周期偏差 | 实际 vs 计划认证周期 | ±15% |
| 证书到期预警率 | 到期前 90 天内更新比例 | 100% |

## 3. ECO 指标

| 指标 | 定义 | 目标 | 说明 |
|:-----|:-----|:-----|:------|
| ECO 密度 | ECO 数 / ProductPlan 数 | 越低越好 | 反映设计成熟度 |
| ECO 周期 | ECO 创建 → 关闭天数 | ≤ 14 天 | 变更效率 |
| ECO 来源分布 | Supplier/Test/Cert/Customer/MFG 占比 | 追踪趋势 | 质量溯源 |
| ECO 影响范围 | 单次 ECO 影响的 Prototype/Project 数 | 越低越好 | 变更管理质量 |

## 4. 治理合规指标

| 指标 | 定义 | 目标 | 频率 |
|:-----|:-----|:-----|:------|
| Architecture Review 通过率 | 评审通过 / 总评审 | ≥ 90% | 每评审 |
| 合规审计评分 | 38 条编码原则通过率 | ≥ 93% | 每次提交 |
| 文档完整率 | 必填字段完成度 | 100% | 实时 |
| 数据质量评分 | P0/P1/P2 门禁通过率 | ≥ 95% | 月度 |

## 5. AI 效能指标

| 指标 | 定义 | 目标 | 说明 |
|:-----|:-----|:-----|:------|
| Agent 任务完成率 | 成功完成 / 总任务 | ≥ 85% | 每 Agent 独立统计 |
| AI 决策采纳率 | 用户采纳 AI 建议 / AI 总建议 | ≥ 70% | 反映 AI 质量 |
| Review Agent 准确率 | 评分与人工评审一致性 | ≥ 80% | 月度校准 |
| Agent 响应时间 P95 | 95% 任务的完成时间 | ≤ 30 秒 | 用户体验 |
| AI 审计覆盖率 | 可追溯的 AI 决策占比 | 100% | 合规要求 |

## 6. 运营指标

| 指标 | 定义 | 目标 | 来源 |
|:-----|:-----|:-----|:------|
| 系统可用性 | (1 - 停机/总时间) × 100 | ≥ 99.9% | 监控系统 |
| P0 事件 MTTR | 平均恢复时间 | ≤ 2 小时 | 事件管理 |
| P0 事件数量 / 月 | 每月 P0 事件数 | ≤ 1 | 事件管理 |
| 变更成功率 | 成功部署 / 总变更 | ≥ 95% | 发布管理 |
| 配置准确性 | CMDB 与实际一致率 | ≥ 98% | 配置审计 |

## 7. KPI 数据血缘

```
ProductPlan
  ├── Idea → Plan 转化率
  ├── Planning → MP 周期
  └── ECO 密度

Verification
  ├── 测试覆盖率
  └── 首次通过率

Prototype
  ├── 阶段回退率
  └── 开发周期

Certification
  ├── 认证通过率
  └── 认证周期偏差

ECO
  ├── 来源分布
  ├── 平均周期
  └── 影响范围
```

## 8. 数据采集要求

| 要求 | 说明 | 优先级 |
|:-----|:------|:-------|
| **自动采集** | KPIs 通过 API 自动统计，禁止手工填报 | P0 |
| **实时更新** | 核心指标（可用性/事件数）实时更新 | P0 |
| **趋势图表** | 所有指标提供 30 天/季度/年度趋势 | P1 |
| **自动告警** | 指标偏离目标自动通知 Owner | P1 |
| **数据导出** | 支持 CSV/JSON 导出 | P2 |
